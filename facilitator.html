First time only - set up a new facilitator
Onboard a new operator onto the joint system →

1. Invite them to the GitHub org (from your machine). Swap in their email:

gh api -X POST orgs/Clinic-Catalyst-AU/invitations -f email=THEIR_EMAIL -f role=direct_member

2. They run the install page top to bottom: open the install page →

3. The 2 things the script can't do (do live): type the SHARED keys into their ~/Systems/BusinessOps/.env in person (never over chat), and connect the MCP connectors (Fireflies, Supabase, GitHub) with their own logins.

If something sticks
Quick fixes →

claude not recognised → close the window, open a fresh one (the command only exists in windows opened AFTER install). Mac: source ~/.zprofile · Windows: $env:Path += ";$env:USERPROFILE\.local\bin"

git asks for a username → gh auth setup-git (already in the bootstrap)

a skill is missing → cd ~/Systems/cc-aios && git pull && bin/apply-to-machine.sh

winget not found (Windows) → install "App Installer" from the Microsoft Store (aka.ms/getwinget), paste the command again

AHPRA worry → every CC skill bakes it in (concern-led, no specialist/expert, no outcome claims)
