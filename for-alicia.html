Part 1 - How the system works (the moving parts)
🧠

1. The Business Brain - the spine

One document per clinic that holds their voice, their offer, and their clients' real emotional truth. It is built first (the resonance interview), and every tool reads it. This is why everything sounds like them and not like generic AI. Lives in their ~/Clinic/Business-Brain/ folder.

⚡

2. The skills - already built, you just run them

Each clinic installs a pack of ready-made skills - content engine, ad builder, follow-up agent, hooks, landing pages, SEO. They read the Business Brain and write finished work. You run one with a command (e.g. /cc-content-engine). You never build them in the room - that is the whole shift from how we've done it.

💻

3. The install - the harness it all runs in

Claude Code + the skill pack + the tools, set up on each machine. It takes 30-60 minutes (mostly downloading), so it runs in the background while you present. It is the one genuinely slow thing, which is why it has to start at 9:00.

🏥

4. GHL - the hands

Where the leads, bookings, automations, the nurture and the AI reception agent all live. Claude writes; GHL sends, books and tracks. The nurture ships as a ready-made GHL snapshot (the workflow + email shells) and the skill just fills the copy in their voice.

Install → Business Brain → run the skills → GHL delivers

That order is the whole game. Nothing produces good output until the Business Brain exists - so foundation always comes first.
